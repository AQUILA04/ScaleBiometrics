# Story 1.3: Multi-tenant Data Isolation

**Status:** Draft

## Story
**As a** Backend Developer,
**I want** to implement Schema-per-Tenant pattern,
**so that** data is strictly isolated.

## Acceptance Criteria
1. **Dynamic Routing**: Route DB queries based on `TenantContext`.
2. **Liquibase**: Master changelog for `public`, separate for tenant schemas.
3. **Provisioning**: Service to create `t_{uuid}` schema and run migrations.
4. **Hibernate**: `MultiTenantConnectionProvider` implementation.

## Tasks
- [ ] Configure Hibernate Multi-tenancy.

- [ ] Implement `TenantService.createTenant()`.
- [ ] Test isolation (Tenant A cannot see Tenant B data).
